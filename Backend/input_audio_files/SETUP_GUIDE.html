🛠️ Setup Guide

Get the Speaker Diarization System running in 5 minutes

1 Requirements

🐍
Python
3.8 or higher
🔊
FFmpeg
For audio conversion
💾
Disk Space
~500MB for models
🧠
RAM
4GB minimum

2 Install FFmpeg

FFmpeg is required for audio format conversion. Choose your method:

Option A: Using winget (Recommended)

winget install FFmpeg

Option B: Using Chocolatey

choco install ffmpeg

Option C: Manual Download

  1. Download from ffmpeg.org
  2. Extract to C:\ffmpeg
  3. Add C:\ffmpeg\bin to your PATH environment variable
✅
Verify Installation: Run ffmpeg -version in terminal

3 Install Python Dependencies

Open terminal in the project folder and run:

pip install faster-whisper librosa pydub scikit-learn numpy soundfile

Or install from requirements.txt:

pip install -r requirements.txt

📦 Dependencies Explained

Package Size Purpose
faster-whisper ~50MB Speech-to-text transcription
librosa ~20MB Audio feature extraction (MFCCs)
pydub ~1MB Audio format conversion
scikit-learn ~30MB Speaker clustering algorithm
soundfile ~5MB WAV file handling

4 First Run (Model Download)

On first run, Whisper models will be downloaded automatically:

python diarize_transcribe.py --input "your_audio.mp3"
⏳
First run takes longer! The Whisper model (~145MB for "base") will be downloaded and cached. Subsequent runs work offline.

📊 Model Download Sizes

Model Download Size Use Case
tiny ~75MB Fast, lower accuracy
base ~145MB ⭐ Default, good balance
small ~465MB Better accuracy
medium ~1.5GB High accuracy
large-v3 ~3GB Best accuracy

5 Usage

🎯 Basic Usage
# Process single file
python diarize_transcribe.py --input "recording.mp3"

# Process all MP3 files in folder
python diarize_transcribe.py --all
⚙️ Advanced Options
# Specify number of speakers (default: 2)
python diarize_transcribe.py --input "audio.mp3" --num_speakers 3

# Use a larger/more accurate model
python diarize_transcribe.py --input "audio.mp3" --model small

# Custom output directory
python diarize_transcribe.py --input "audio.mp3" --output ./results

📋 All Command Line Options

Option Type Default Description
--input, -i Required* - Input audio file path
--all, -a Required* - Process all MP3s in folder
--output, -o Optional ./output Output directory
--num_speakers, -n Optional 2 Number of speakers
--model, -m Optional base Whisper model size

* Either --input or --all is required

6 Output Files

After processing, you'll find these files in the output/ folder:

📁 output/
    📄 recording_transcript.txt   ← Human-readable
    📄 recording_diarized.json   ← Structured data
    📄 recording.srt   ← Subtitles

🔧 Troubleshooting

❌ "FFmpeg not found"

FFmpeg is not installed or not in PATH. Reinstall using Step 2.

❌ "No module named 'faster_whisper'"

Dependencies not installed. Run: pip install faster-whisper

❌ "CUDA out of memory"

GPU memory issue. The script defaults to CPU, which should work fine. If issues persist, try a smaller model: --model tiny

⚠️ Slow processing

Normal on CPU. A 4-minute audio takes ~30-45 seconds. For faster results, use --model tiny.
