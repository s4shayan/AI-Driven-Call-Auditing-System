1Audio Preprocessing

The first step converts the input audio to a standardized format for processing:

def convert_to_wav(input_path, output_path):
    audio = AudioSegment.from_file(input_path)
    audio = audio.set_frame_rate(16000).set_channels(1)
    audio.export(output_path, format="wav")

3Feature Extraction (The Core)

This is the heart of speaker separation. For each segment, we extract acoustic features unique to each speaker's voice:

🎵
MFCCs (20 coefficients)
Mel-Frequency Cepstral Coefficients capture vocal tract shape, voice timbre, and speaking characteristics - unique to each person.
☀️
Spectral Centroid
Measures voice "brightness" - the center of mass of the spectrum. Higher values = brighter voice.
📶
Spectral Bandwidth
Voice frequency spread - varies significantly between speakers based on vocal characteristics.
〰️
Zero Crossing Rate
Speech/silence patterns and voice texture - helps distinguish different speaking styles.

How MFCCs Work

Audio Signal
→

→

→

→

20 MFCCs
Why MFCCs distinguish speakers: Each person's vocal tract has unique physical properties (throat length, nasal cavity size, etc.) that create distinctive spectral patterns captured by MFCCs.

Final Feature Vector

Each segment gets a 43-dimensional feature vector:

[MFCC_mean(20) + MFCC_std(20) + spectral_centroid(1) + spectral_bandwidth(1) + ZCR(1)]

4Speaker Clustering

Groups segments by voice similarity using Agglomerative Clustering:

# Normalize features
scaler = StandardScaler()
features_normalized = scaler.fit_transform(features)

# Cluster into speakers
clustering = AgglomerativeClustering(
    n_clusters=num_speakers,  # Default: 2
    metric='euclidean',
    linkage='ward'
)
labels = clustering.fit_predict(features_normalized)

How Agglomerative Clustering Works

StageProcess
InitialEach segment is its own cluster: [Seg1] [Seg2] [Seg3] [Seg4] [Seg5]
Step 1Merge most similar: [Seg1+Seg2] [Seg3] [Seg4+Seg5]
Step 2Continue merging: [Seg1+Seg2+Seg3] [Seg4+Seg5]
FinalSPEAKER_00: Seg1,2,3 SPEAKER_01: Seg4,5
Ward Linkage: Minimizes variance when merging clusters - produces compact, well-separated speaker groups.

5Post-Processing

5.1 Temporal Smoothing

Fixes unrealistic rapid speaker switching by reassigning short isolated segments:

# If a short segment (<1s) is surrounded by same speaker, reassign it
if seg_duration < min_duration:
    if labels[i-1] == labels[i+1] and labels[i] != labels[i-1]:
        smoothed[i] = labels[i-1]
Example
BeforeSPEAKER_00 (3s) → SPEAKER_01 (0.5s) → SPEAKER_00 (2.5s) → SPEAKER_00 (4s)
AfterSPEAKER_00 (3s) → SPEAKER_00 (0.5s) → SPEAKER_00 (2.5s) → SPEAKER_00 (4s)

5.2 Merge Consecutive Segments

Combines adjacent segments from the same speaker for cleaner output:

if seg["speaker"] == merged[-1]["speaker"]:
    merged[-1]["end"] = seg["end"]
    merged[-1]["text"] += " " + seg["text"]
